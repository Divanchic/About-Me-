Контрольные вопросы
1.	Что такое DLL и для чего она используется? - DLL (Dynamic Link Library) — это динамически подключаемая библиотека, которая содержит код, данные и ресурсы, доступные для нескольких программ одновременно. DLL позволяет разделять функционал между приложениями, уменьшать размер программ и обеспечивать повторное использование кода.

2.	Какие основные преимущества дает использование DLL в программировании? - Преимущества использования DLL
•	Модульность. Позволяет разбить программу на отдельные компоненты.
•	Экономия памяти. Общий код загружается в память один раз и используется разными приложениями.
•	Обновляемость. Можно обновлять части функционала, не меняя всю программу.
•	Повторное использование кода. Одинаковые функции не нужно дублировать в каждой программе.

3.	В чем разница между статической и динамической загрузкой DLL? - 
•	Статическая загрузка: ссылка на DLL определяется на этапе компоновки, и библиотека загружается автоматически при запуске приложения.
•	Динамическая загрузка: приложение самостоятельно загружает DLL во время выполнения с помощью функций, например, LoadLibrary или dlopen (в Unix-системах).

4.	Как экспортируются функции из DLL? - Для экспорта используется модификатор __declspec(dllexport) в Windows или соответствующие атрибуты в других ОС.

5.	Что такое импорт функций из DLL и как он реализуется? - Импорт функций — приложения вызывают экспортированные функции из DLL.Для импорта — __declspec(dllimport) или явное использование API функций.

6.	Как на Windows осуществляется динамическая загрузка DLL? - При динамической загрузке приложение получает дескриптор DLL, а затем запрашивает адрес нужной функции с помощью GetProcAddress.

7.	Какие функции Windows API используются для загрузки DLL и получения адреса функции? - При загрузке DLL операционная система связывает вызовы функций с адресами внутри DLL.

8.	Какие типичные ошибки могут возникать при работе с DLL и как их обрабатывать? - 
•	Возможные ошибки: отсутствие DLL, неуспешная загрузка, отсутствующая экспортируемая функция.
•	Необходимо выполнять проверку возвращаемых значений функций загрузки и получения адресов

9.	Приведите примеры практического использования DLL в программировании. - 
•	Универсальные библиотеки для работы с графикой, сетью, математикой.
•	Плагины для программ (расширения функционала без перекомпиляции).
•	Обеспечение совместного доступа к аппаратным ресурсам и драйверам.

10.	В чем преимущества использования DLL при разработке плагинов и расширений для приложений? -
•	Модульность. Позволяет разбить программу на отдельные компоненты.
•	Экономия памяти. Общий код загружается в память один раз и используется разными приложениями.
•	Обновляемость. Можно обновлять части функционала, не меняя всю программу.
